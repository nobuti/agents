Prescription · decide before the first commit

  • Peel T1 off the front. A behavior-preserving extraction never rides with the feature that motivated it — standalone → main first, approved in minutes.
  • Route T2–T5 through an integration branch. A flagless atomic cutover can't merge half-done to main; the integration branch buys small reviews without a feature flag.
  • Pre-budget the tests. T2/T3 forecast as test-heavy — most of the diff will be soft tissue. Plan the review around the prod skeleton, not the line count.
  • Re-scan if this grows. If editing adds a 4th domain or a 2nd parallel branch appears, the verdict flips to split the SPEC — back to spec phase, don't push through.
